# 手风琴入门应用：产品与技术全方案

> 创建日期：2026-05-19

## 一、产品方向

### 核心定位

**"手风琴版的钢琴键盘玩具"** —— 降低手风琴学习门槛，用亮灯引导取代传统识谱，让零基础用户 10 分钟内就能"演奏"一首歌。

### 用户层次（三圈模型）

```
外圈：B 端（厂商授权 + UP 主内容生态）
中圈：进阶学习者（有意识学习、购买课程）
内圈：入门用户（老年人、零基础、买了琴不会玩的人）
```

| 用户画像 | 核心诉求 | 设计要点 |
|---------|---------|---------|
| 60+ 老年人 | 能玩、不难、有成就感 | 大字体、慢节奏、多鼓励 |
| 零基础年轻人 | 快速上手、看起来炫 | 游戏化、社交分享 |
| 手风琴 UP 主 | 制作自己的教学关卡 | 创作者工具、流量入口 |
| 手风琴厂商 | 产品增值服务 | 白标 SDK、换肤、绑定琴型 |

### 核心功能模块

```
1. 虚拟手风琴
   ├── 右手键盘（钢琴键）可视化 + 亮灯
   ├── 左手按钮（Stradella 系统）可视化 + 亮灯
   └── 按下动效 + MIDI 发音

2. 教学引擎
   ├── Demo 模式：自动播放，看灯看谱
   ├── 跟练模式：亮灯等你按，按对才进行
   └── 自由演奏模式：不限制，随便玩

3. 关卡系统
   ├── 新手村（单手、简单旋律）
   ├── 进阶（双手协调）
   └── 挑战曲目

4. 内容创作工具（Phase 2）
   ├── 谱子编辑器（导入 MusicXML/MIDI）
   └── 关卡打包发布
```

### 商业路径

- **Phase 1**：免费跑通口碑，积累用户
- **Phase 2**：课程内购 + UP 主创作者生态
- **Phase 3**：厂商白标授权（预装到琴机里，或配套 App）

## 二、技术路线

### 跨平台策略选择

最关键的决策：**起点是微信小程序，同时覆盖 Web，后续扩展 iOS/Android 原生**。

| 方案 | 小程序支持 | Web(H5) | iOS/Android | 学习成本 | 推荐度 |
|-----|----------|---------|-------------|---------|-------|
| Taro 3 (React) | ★★★★★ | ★★★★★ | ★★★★ (via RN) | 中 | ✅ 推荐 |
| Uni-app (Vue) | ★★★★★ | ★★★★ | ★★★★ | 低 | 次选 |
| Flutter | ★★ (第三方) | ★★★ | ★★★★★ | 高 | 不适合 |
| React Native | ★ | ★ | ★★★★★ | 中 | 不适合 |

**结论：Taro 3 + React + TypeScript**

理由：
- 专为"小程序优先、多端兼容"设计
- Taro 内置 `h5` 编译目标，同一套代码一行配置出 Web 版
- React 生态全球最成熟，未来招人容易
- 同一套代码可编译到小程序、H5、React Native
- TypeScript 强类型对后续扩展至关重要

**平台矩阵（同一个 core）：**

```
@accordion/core（纯 TS）
├── Taro 编译 → 微信小程序
├── Taro 编译 → H5 网页（轻量版，可嵌入厂商官网）
├── Taro 编译 → React Native → iOS / Android
└── Next.js   → 创作者平台 / 厂商定制网页（完整 Web 版）
```

### 核心数据模型

这是整个架构最重要的部分，定义好了后续扩展非常容易：

```typescript
// 1. 琴型配置（换品牌/换琴型只需换这个JSON）
interface AccordionConfig {
  id: string;                    // "piano-96-bass-hohner"
  name: string;                  // "Hohner 96贝司钢琴手风琴"
  manufacturer?: string;
  type: 'piano' | 'button-chromatic' | 'button-diatonic' | 'bayan';

  treble: {                      // 右手（高音侧）
    keys: TrebleKey[];           // 每个键的位置、MIDI音符号
    layout: 'piano' | 'chromatic-c' | 'chromatic-b';
  };

  bass: {                        // 左手（低音侧）
    buttons: BassButton[];       // 每个按钮的位置、功能
    system: 'stradella' | 'free-bass' | 'converter';
    rows: number;                // 行数（通常6行）
    cols: number;                // 列数（通常16/20列）
  };

  visual: {
    bodyAsset: string;           // 琴身图片/SVG
    keyAssets: { normal: string; pressed: string };
    buttonAssets: { normal: string; pressed: string };
    highlightColor: string;      // 亮灯颜色（可品牌化）
  };
}

// 2. 课程/关卡格式（UP主上传的核心格式）
interface Course {
  id: string;
  title: string;
  author: string;
  targetAccordionTypes: string[];  // 适配哪些琴型
  difficulty: 1 | 2 | 3 | 4 | 5;
  levels: Level[];
}

interface Level {
  id: string;
  title: string;
  bpm: number;
  mode: 'demo' | 'guided' | 'freeplay';
  events: LessonEvent[];         // 时间轴事件流
}

interface LessonEvent {
  time: number;                  // ms时间戳
  type: 'note' | 'chord' | 'hint' | 'lyric';

  notes?: {
    hand: 'left' | 'right';
    keys: string[];              // 对应AccordionConfig中的key id
    duration: number;
    midi: number;
  }[];

  hint?: string;                 // 提示文字
  lyric?: string;                // 歌词
}
```

**这个格式的优势：**
- **一份谱子，多种琴型**：通过 `keys` 引用逻辑键名，渲染层按琴型配置映射到实际位置
- **UP主友好**：可从 MusicXML/MIDI 自动转换生成
- **厂商可定制**：只需换 `AccordionConfig` 里的 `visual` 字段

### 音频方案

微信小程序没有 Web Audio API / MIDI API，需要用预渲染采样：

```
方案：SoundFont 采样包（每个音符一个 MP3）
├── 在构建时将 SoundFont 转换为 MP3 采样文件
├── 按需加载（CDN）
├── 微信小程序：InnerAudioContext 并发播放
├── Web(H5)：Web Audio API（AudioContext + BufferSource）
└── React Native：expo-av / react-native-sound
```

> Web Audio API 支持多声部同时播放、精确时序控制，比小程序环境能力更强，H5 版音频体验反而是最好的。

音频抽象层接口：

```typescript
// packages/core/src/audio/AudioEngine.ts
interface AudioEngine {
  loadInstrument(soundfont: string): Promise<void>;
  playNote(midi: number, duration: number): void;
  stopNote(midi: number): void;
  setTempo(bpm: number): void;
}

// 各平台实现各自的 adapter
// miniprogram/src/adapters/WeixinAudioEngine.ts  — InnerAudioContext
// web/src/adapters/WebAudioEngine.ts             — Web Audio API
// mobile/src/adapters/ExpoAudioEngine.ts         — expo-av
```

### 虚拟手风琴渲染

用 Canvas 渲染（跨平台一致性最好）：

```
渲染层设计：
├── 背景层：琴身图片（静态）
├── 键盘层：右手钢琴键（SVG 或 Canvas）
├── 按钮层：左手按钮阵列（Canvas 绘制，数学坐标定位）
├── 高亮层：亮灯动效（Canvas 动画）
└── 交互层：触摸事件捕获
```

> 注意：左手按钮是六排斜列阵列，用数学坐标系定位每个按钮，而非图片切图，否则每个品牌都要重新处理资源。

### 关卡引擎状态机

```
IDLE → LOADING → READY → PLAYING → WAITING_INPUT → CHECKING →
                                                      ├── CORRECT → PLAYING（下一个）
                                                      └── WRONG → FEEDBACK → WAITING_INPUT

模式差异：
- Demo 模式：PLAYING 中不等待输入，自动播放
- Guided 模式：到每个标记点进入 WAITING_INPUT
- Freeplay 模式：始终在 PLAYING，检测输入但不阻塞
```

## 四、扩展点总结

| 扩展场景 | 如何支持 |
|---------|---------|
| 换手风琴品牌 | 替换 `AccordionConfig` JSON + 视觉资源 |
| 加新琴型（巴扬） | 新增一个 `AccordionConfig`，core 层零改动 |
| UP 主上传谱子 | 提供 MusicXML/MIDI → `LessonEvent[]` 转换工具 |
| 发布 Web 版（轻量） | `taro build --type h5`，无需改代码 |
| 发布 Web 版（完整） | Next.js 复用 core + 组件，独立部署 |
| 迁移 iOS/Android | Core 层零改动，只需实现平台 Adapter |
| 厂商白标 | 换主题配置 + `AccordionConfig` + CDN 资源，可单独给 H5 版 |
| 蓝牙实体琴 | 在 Adapter 层增加 BLE MIDI 驱动 |
